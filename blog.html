(1) Discuss the scope of var, let, and const

var: Function-Level Scope: Variables declared with var are function-scoped, which means they are visible throughout the entire function in which they are declared. This behavior can sometimes lead to unexpected results, especially inside loops or conditional statements.
Hoisting: var variables are hoisted to the top of their containing function or global scope during the compilation phase. This means that you can access and use a var variable before it's declared in the code, but it will have an initial value of undefined.
Global Scope: If declared outside of any function, var variables become global variables and can be accessed from anywhere in the code, which can lead to potential naming conflicts and unexpected side effects.

(2) Tell us the use cases of null and undefined

Null:
(1)Represents the intentional absence of a value.
(2)Can be assigned to variables or properties to indicate an empty or unset state.
(3)Returned by functions to signify a successful operation with no meaningful result.
Undefined:
(1)Automatically assigned to variables that are declared but not initialized.
(2)Commonly encountered when accessing object properties or variables that don't exist.
(3)Often used as a default value for function parameters.

(3) What do you mean by REST API?

Certainly! One unique aspect of RESTful APIs is the use of standard HTTP methods (GET, POST, PUT, DELETE, etc.) to perform operations on resources. These methods map to common CRUD (Create, Read, Update, Delete) operations in a predictable and intuitive way:
GET: Used to retrieve data from a resource. It should not have any side effects on the resource.
POST: Used to create a new resource. It often involves sending data to the server to be processed and stored.
PUT: Used to update an existing resource or create it if it doesn't exist. It typically replaces the entire resource with the new data.
DELETE: Used to remove a resource from the server. These HTTP methods provide a uniform and consistent way to interact with resources, making RESTful APIs easy to understand and work with. It's one of the key design principles that sets REST apart from other architectural styles for building APIs.
